# Usage — TaskChute Mini

このドキュメントは、TaskChute Mini を  
**今日から迷わず使うための実践ガイド**です。

すべての機能を理解する必要はありません。  
まずは「開始して、終わる」ことだけできれば十分です。

---

## TaskChute Mini が目指していること

TaskChute Mini は、

- 「今日をきれいに終わらせる」ためのツールではありません
- 「今日を効率的に過ごす」ためのツールでもありません

今やっていることに意識を戻し、  
止まっても・ズレても、  
**そのまま続きを書ける状態に戻る**ための実行ログです。

---

## クイックスタート（5分）

### 1. 今日のログを開く

次のいずれかを実行します。

- コマンド  
  TaskChute: Open Today
- Player Mode の Today ボタン
- Chute View から自動で対象ログを読込

logFolderPath/YYYY-MM-DD.md が開きます  
（存在しない場合は自動で作成されます）

---

### 2. タスクを書く

トップレベルの `- ` 行はすべてタスクです。

例：

- メール確認
- 企画書を読む
- 会議

順番・粒度・正確さは気にしなくてOKです。

---

### 3. タスクを開始する

カーソルを親タスク行に置いて、以下のいずれかを実行します。

- Start
- Insert & Start（タスクがまだ無い場合）

実行すると、タスク直下に ⌛ 行が追加されます。

例：

- メール確認  
  - ⌛ 09:10–

この ⌛ 行が「今やっていること」です。

---

### 4. タスクを終了する

作業が終わったら：

- End

例：

- メール確認  
  - ✔️ 09:10–09:25 +15m

時間は自動で計算されます。  
多少ズレても問題ありません。

---

### 5. 次に進む

- End & Start

今のタスクを終了し、  
次の未処理タスクを自動で開始します。

---

## Chute View（フルページ）

Chute View は、**ログを開かずに操作できる**フルページUIです。

### 開き方

- TaskChute: Open Chute View (Full Page)

### できること

- Start / End / End & Start / Time / Add
- タスクリストをタップして開始（Ready のときのみ）
- Multiple ⌛ の検知と修復

### Start の挙動

- Now がある時：開始不可（Notice）
- Ready の時：
  - タスクリストをタップ → 確認 → 開始
  - Start ボタン → 「上から開始」確認 → 開始

---

## ログの基本ルール

### 親タスク

- トップレベルの `- ` 行はすべてタスク
- 並び順が優先度・実行順になります

---

### 実行ログ（子行）

- 実行中  
  - ⌛ HH:mm–

- 完了  
  - ✔️ HH:mm–HH:mm +Xm

子行は親タスクの直下に付きます。

---

### メモ

- 📝 気づいたこと

- タスク直下に書く簡単なメモ
- 整理や清書は不要です

---

## Player Mode（下部操作パネル）

### 表示条件

- taskchute ログを開いている
- モバイルでキーボードが閉じている

### できること

- Start / End / End & Start
- 上下移動（行フォーカス）
- Player ⇄ Grid 切り替え
- Focus / Filter の切り替え

考えずに押せる操作だけを集めています。

---

## Techo（月次）から今日分をインポート

### コマンド

- TaskChute: Import Techo Today

### 事前設定

- Techo folder path（`YYYY-MM.md` が入っているフォルダ）
- Techo import header（追加する見出し）
- Techo import mode（Append / Replace）

### 挙動

- 今日の日付に対応する `## M月D日(...)` の配下から `- ` を抽出
- TaskChute ログの `## techoからインポート` 配下へ追加
- 2回以上実行しても同一行は増殖しない（append）

## Player/Horizon の配置とサイズ

### 表示位置

- Player placement で Embedded / Floating を切り替え
- Floating の場合はドラッグ移動可能
  - Player は下段中央のグリップバー
  - Horizon はヘッダー全体を掴む

### サイズ

- Player size / Horizon size で拡大縮小（70–140%）
- Size range を変更するとスライダー範囲も変わる

## 表示モード

### Focus Mode

- 親タスクは表示
- 子タスクは ⌛ 行のみ表示
- それ以外は非表示（本文は変更しない）

→ 今やっていることだけを見るためのモード

---

### Filter Mode

- ✔️ を含むタスクブロックを非表示

→ まだ手を付けていないタスクだけを見るモード

---

### Dim（今に集中）

- 全体の文字色を薄く表示
- 実行中タスクの親行のみを強調

→ 「どこを見ればいいか」を視覚的に示します

---

## 見積もり（任意）

親タスクにそのまま書けます。

例：

- 企画書を読む (20m)

- (20) / (20m) どちらもOK
- 見積は補助情報です
- 正確である必要はありません

Player / Header では、

- 残り見積時間
- すべて終わる予想時刻（ETA）

を確認できます。

---

## テンプレート機能

### テンプレートの考え方

- テンプレは「作業の型」
- フォルダ内の Markdown を使用
- ## セクション単位で挿入します

用途例：

- 平日 / 休日
- 午前 / 午後
- ルーチン

---

### 複数テンプレの挿入

- 候補を 1 つずつ選択
- 選んだものは次回候補から除外
- Exit を選ぶまで追加可能

選択した順番のまま、ログ末尾に挿入されます。

---

## 前日のタスクを使う

Copy From Previous Day で：

- 前日すべて
- 未実行タスクのみ
- セクション選択

をコピーできます。

「昨日の続きから始める」ための機能です。

---

## 迷ったときの指針

- 書けなくても Start だけ押していい
- 見積は外れて当たり前
- ログが雑でも問題ない
- 途中でやめても、明日また開けばいい

TaskChute Mini は  
完璧な一日を作るためのツールではありません。

今日を、  
**また続けられる状態に戻すための道具**です。
